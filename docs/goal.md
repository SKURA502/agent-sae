# 工具调用机制可解释性

## 研究问题

> LLM 内部的哪些表征决定了是否调用工具？

## 三个核心假设

**H1 — 工具调用门控**：残差流中的一组稀疏 latent features 在 action boundary 处充当工具调用决策的门控信号，这组特征集中、线性可读，并能跨任务类型泛化。

**H2 — 证据积累**：门控随上下文中感知到的信息缺口持续积累，而非单点触发。工具成功返回后门控被抑制；空返回或错误返回则将其放大。

**H3 — 因果可控性**：对门控 features 的定向激活干预能可靠地改变工具调用行为（决策翻转率 > 20%），且不显著损害语言质量（perplexity 变化 < 20%）。

## 方法

- **模型**：Qwen3.5-4B（主）、Qwen3.5-9B（规模对比）
- **表征工具**：TopK 稀疏自编码器（SAE），作用于模型约 3/4 和 5/6 深度处的残差流
- **SAE 训练**：两阶段——Stage 1 通用文本（OpenWebText2，~50M tokens），Stage 2 工具调用域微调（When2Call Pref，~6K 平衡样本）
- **因果证据**：对 top 门控 features 做激活 steering / ablation，测量决策翻转率
